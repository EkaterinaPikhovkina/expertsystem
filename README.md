# TechSupport Bot

Telegram-бот технічної підтримки для інтернет-провайдера. Автоматично діагностує мережеві проблеми (відсутність інтернету, низька швидкість) на основі даних з мережевого обладнання (OLT) та білінгу — без залучення оператора.

## Швидкий старт

### 1. Клонувати / перейти до проєкту

```bash
cd techsupport_bot
```

### 2. Створити та активувати віртуальне середовище

```bash
python3 -m venv .venv
source .venv/bin/activate   # Linux / macOS
.venv\Scripts\activate      # Windows
```

### 3. Встановити залежності

```bash
pip install -r requirements.txt
```

### 4. Створити базу даних PostgreSQL

```bash
psql -U postgres -c "CREATE DATABASE techsupport;"
psql -U postgres -d techsupport -f database/schema.sql
```

### 5. Отримати токен бота

1. Відкрити Telegram → знайти **@BotFather**
2. Надіслати `/newbot` та слідувати інструкціям
3. Скопіювати виданий токен вигляду `7123456789:AAF...`

### 6. Налаштувати змінні середовища

```bash
cp .env .env
```

Відредагувати `.env`:

```env
BOT_TOKEN=7123456789:AAF...

DB_HOST=127.0.0.1
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=ваш_пароль
DB_NAME=techsupport
```

### 7. Запустити бота

```bash
python3 bot.py
```

Успішний запуск виглядає так:

```
INFO: Database pool created
INFO: Starting polling...
```

### 8. Запустити тести (опційно)

```bash
python3 -m pytest tests/ -v
```

## Функціональність

### Авторизація

При старті (`/start`) бот пропонує ввести **номер договору** або **адресу**. Після ідентифікації перевіряється стан рахунку:

| Стан | Поведінка бота |
|---|---|
| Баланс від'ємний | Повідомлення про борг, доступ заблоковано |
| Договір заморожено | Пропозиція активувати послугу |
| Все ОК | Перехід до головного меню |

---

### Головне меню

```
🔴 Немає інтернету
🐢 Низька швидкість
💳 Мій рахунок
❄️ Розморозка договору
```

---

### Діагностика «Немає інтернету»

**Етап 1 — Аналіз обладнання (OLT):**

Система зчитує з бази даних імітовані параметри оптичного терміналу (ONU) і аналізує:

| Параметр | Що перевіряється |
|---|---|
| Статус ONU | Онлайн / офлайн |
| MAC-адреси | 0 — немає зв'язку з роутером; 1 — норма; 2 — перезавантажити роутер; >2 — петля (кабель у LAN замість WAN) |
| Швидкість порту | full-1000 / full-100 / full-10 відносно тарифу |
| Рівень сигналу | Норма / слабкий (числа користувачу не показуються) |
| Трафік | Є обмін даними чи ні |
| Останні події | DyingGasp (живлення), LOSi (оптика) |

**Етап 2 — Опитування по індикаторах (якщо ONU офлайн):**

Бот послідовно запитує про стан індикаторів **PWR → PON → LOS** і на основі відповідей видає висновок:

| Ситуація | Висновок |
|---|---|
| PWR не горить / мигає | Проблема з блоком живлення |
| LOS горить червоним | Пошкоджено оптичний кабель/конектор |
| PON мигає, LOS не горить | Термінал намагається авторизуватись |

---

### Діагностика «Низька швидкість»

Виконується той самий аналіз OLT, плюс надаються рекомендації:

- Пояснення різниці між Wi-Fi 2.4 ГГц і 5 ГГц
- Тест швидкості при підключенні кабелем напряму
- Перевірка навантаження від інших пристроїв у мережі

---

### Система тікетів

- Після кожної діагностики бот питає: **«Проблему вирішено?»**
- Якщо **Так** — тікет закривається
- Якщо **Ні** — діагностика повторюється
- Якщо при повторній діагностиці результат **збігається** з попереднім — автоматично створюється заявка для технічного відділу

---

### Розморозка договору

Бот показує суму місячної абонплати і після підтвердження імітує оплату — статус договору змінюється на «активний».

## Тестові дані

Після виконання `schema.sql` у базі є 5 клієнтів:

| Договір | Адреса | Ситуація |
|---|---|---|
| `100001` | вул. Шевченка 1, кв. 5 | Активний, ONU онлайн, тариф 100 Мбіт/с, все норма |
| `100002` | вул. Лесі Українки 22 | Борг (баланс −50 грн) → доступ заблоковано |
| `100003` | пр. Незалежності 7, кв. 3 | Договір заморожено → flow розморозки |
| `100004` | вул. Садова 14 | Тариф 500 Мбіт/с, лінк full-10 → критична швидкість |
| `100005` | вул. Центральна 3, кв. 11 | ONU онлайн, full-100 при тарифі 100 Мбіт/с |

## Структура проєкту

```
techsupport_bot/
├── bot.py                  # Точка входу
├── config.py               # Налаштування з .env
├── states.py               # FSM-стани розмови
├── keyboards.py            # Inline-клавіатури
├── texts.py                # Всі тексти повідомлень (українська)
├── database/
│   ├── connection.py       # Пул з'єднань asyncpg
│   ├── queries.py          # Всі SQL-запити
│   └── schema.sql          # DDL + тестові дані
├── engine/
│   ├── parser.py           # Regex-парсер виводу ZTE OLT
│   ├── rules.py            # Inference Engine (правила A–G)
│   └── simulator.py        # Генератор випадкових даних ONU
├── handlers/
│   ├── auth.py             # Авторизація
│   ├── menu.py             # Головне меню
│   ├── diagnostics.py      # Діагностика (немає інтернету / низька швидкість)
│   └── account.py          # Рахунок, розморозка
├── tests/
│   └── test_engine.py      # 36 unit-тестів парсера та правил
├── requirements.txt
└── .env.example
```
